## 2. Model Quality & ML
- ◧ **P1** Diagnose the sub-breakeven win rate per sport × stat — IN PROGRESS via `holdout_report`. Finding: NBA points was overconfident (pred 71% → real 46%), but all 79 picks were **playoffs** (calibrator is regular-season-only, `< 2026-01-01`) = distribution shift, not a calibrator bug; refitting on a non-recurring playoff sample would overfit. Fix shipped: **stat-level suppression** in `category_cutoffs` — a (sport,stat) confidently below breakeven (Wilson upper bound) is now suppressed instead of inheriting the more permissive sport cutoff (NBA points → suppressed; NBA rebounds stays tuned). Self-corrects when regular-season data returns.
- ☑ **P1** Confidence threshold tuning — DONE: per-category cutoffs (`props/models/category_cutoffs.py` + `category_cutoffs.json`) auto-derived from settled history as the lowest `model_prob` whose Wilson-LB win rate clears the 57.7% breakeven. Dashboard recomputes from the DB every 6h; recompute the seed offline with `python -m props.models.category_cutoffs`. (MLB ≥0.55 / 64% hist; NBA tuned ≥0.725 once it had enough settled data — see below.) **Refinement (DONE):** per-*stat* cutoffs now *step up* off the volume-max floor when a higher cutoff is materially safer (Wilson-LB higher by > `LB_STEPUP_MARGIN`=0.04) — the daily backtest surfaced MLB hits winning 70% @0.55 vs 84% @0.625, and on a 2-pick parlay that compounds (49%→71% joint), so the tuner now lands MLB hits at 0.625. The coarse sport-level fallback stays permissive (lowest-clearing) so stats inheriting it aren't over-suppressed.
- ☐ **P1** Calibration coverage: only some models have `_calibrator.pkl` (NBA pts/reb/ast yes; threes, MLB HR, all NHL, all WNBA no). Add isotonic calibration everywhere.
- ☑ **P2** Model versioning/registry — DONE (lightweight): `docs/MODEL_VERSIONING.md` defines naming, when to bump `v{N}`, evidence-driven retrain cadence (tied to `holdout_report`), the live cutoff guardrail, and one-commit rollback.
- ☑ **P2** Feature-leakage audit — DONE, clean: every rolling/windowed feature across NBA/WNBA/NHL/MLB (rolling, advanced_stats, basketball_iq, opposing-team/pitcher/lineup, home/away, streak, back-to-back) computes on `.shift(1)` first, so the current game never enters its own features; cross-game lookups (`inference.py`, `nba_teammate_absence`) use strict `< game_date`. Only gray area: `nba_play_types` uses season-level *style* rates (iso/pnr/spot-up tendencies) — descriptive, not the prediction target, not outcome leakage. Added a real-code regression test (`test_rolling_features_have_no_lookahead`) that fails if a `shift(1)` is ever dropped.
- ☐ **P3** Train NHL/WNBA **winner** models once data is sufficient (WNBA first — basketball-generic; revisit ~150+ games). NHL currently 11 games, WNBA 43.
- ☑ **P3** Correlation-aware parlay builder — DONE: `build_diversified_parlay` selects highest-confidence legs while never taking two from the same (game, direction) — the toxic stack that busts as a block when a game runs hot/cold (e.g. Jun-10's pile of same-game unders). Wired into the dashboard slate card (shows "diversified across N games · joint %") and the Discord best-2-pick. (The older `build_parlays` correlation-*exploit* functions remain for the dormant predict path.)

## 5. Evaluation & Tracking
- ☐ **P1** Ensure every retrain logs a `backtest_runs` row (Performance tab already charts it).
- ☑ **P2** Out-of-sample holdout/calibration report — DONE: `python -m props.models.holdout_report` (per sport×stat win rate, predicted-vs-realized calibration + weighted MAE, recent-vs-earlier drift).
- ☑ **P2** Probability recalibration (over-confidence) — DONE: settled picks were over-confident above ~0.60 (0.80–0.90 band predicted 83% but won 62%) because the per-stat isotonic was fit on regular-season data while the live slate drifted (playoff NBA). `props/models/prob_calibration.py` learns a 2-param **Platt** correction (`sigmoid(a·logit(p)+b)`, `a≈0.65` shrinks the inflated extremes) from the system's OWN settled history, recomputed from the live DB with a committed JSON seed + identity fallback (same self-tuning pattern as the cutoffs; Platt over isotonic because the upper tail is thin). Applied where *magnitude* matters — Kelly sizing (`log_picks`), and the confidence shown on the cards / Discord digest / slate joint — while **selection stays on the raw prob + empirical cutoffs** (a monotonic transform doesn't change ordering). Brier 0.245→0.238; an "84%" pick now honestly reads ~72%. The daily backtest reports raw→recalibrated Brier so the correction is monitored.
- ⚠️ **Per-sport market-disagreement filter — REVERTED (was based on contaminated data).** Shipped then reverted same day: the "market_edge" it keyed on turned out to be the **0.5 neutral prior** (`model_prob − 0.5`), not a real model-vs-market gap — every settled pick's stored `market_edge` had `market_implied == 0.5` because the live feed wasn't matching players / the `market_odds` table didn't cover those games at log-time. Re-checked against the **real** `market_odds` rows (94 NBA picks): the >15% true-disagreement bucket actually wins **55.4%**, not 45% — so the filter was dropping *profitable* picks. The 45% "finding" was just high-`model_prob` NBA losing to over-confidence, already handled by the Platt recalibration. Lesson logged: validate market signals against the `market_odds` table, never the stored `market_edge`, until real odds are confirmed flowing.
- ☑ **P2** Model–market blend (real data) — DONE (4 stages, 2026-06-13). `picks.model_prob` now stores a per-sport blend `w·model + (1−w)·market` of the model and the sharp no-vig market, so selection, cutoffs, calibration, and display all use it (raw kept in `model_prob_raw`, market in `market_prob`; migration 0008). Weights self-tune from real-line settled picks (`props/models/blend_weights.py`, JSON seed + pure-model fallback): **NBA w≈0.15** (the deep market dominates) and **MLB w≈0.75** (the model already beats the soft MLB market). Stage 1: extended the live odds path (`market_odds.py`) to MLB (was NBA-only). Stage 3: `blend()` returns the pure model prob when there's no real line — **never** blends on the 0.5 prior (the trap from the reverted filter). Backfilled history (`backfill_blend.py`, 455/862 picks have a line). Result: daily-backtest **Brier 0.245→0.235**, and Platt recalibration fell to near-identity (`a` 0.65→1.0) because the blend already carries the well-calibrated market. *Nuance:* with most picks still lacking a live line, the shared Platt fit now under-corrects the remaining raw (no-market) picks — self-resolves as live market coverage grows (now NBA+MLB). See [[market-edge-is-neutral-prior]].
- ☑ **P2** ROI by parlay size — DONE: Performance tab shows 2/3/4-pick power-play ROI at the recommended-tier per-leg win rate.
- ☑ **P2** Closing Line Value (CLV) — DONE: `compute_clv` captures the last pre-game *standard*-variant line (migration `0005` adds `picks.line_close`), wired into daily.sh; Performance tab shows beat-the-close %, avg CLV, and the +CLV-vs-−CLV win split. Finding: PrizePicks standard lines are sticky (only ~9% move), so CLV is a weaker signal here than at a sharp book — read it as a trend.
- ☑ **P2** Nightly Discord scorecard — DONE: `props/picks/scorecard.py` (daily.sh step 7c) auto-posts last night's results each morning — recommended-tier W/L vs the 57.7% breakeven, per-sport, 7-day rolling, and the weakest stat bucket. No session/manual check needed.

## 9. Autonomous Operations (the system runs/fixes itself)
- ☑ **P0** Cloud scrape (Option B) — DONE: `fetch_projections` routes through `PRIZEPICKS_PROXY` (residential, IPRoyal) so the scrape runs on GitHub Actions despite PrizePicks blocking datacenter IPs; retries cycle past Cloudflare-flagged pool IPs (~⅔ 403). The Mac cron is now an optional backup, not a dependency. Verified end-to-end from a GHA datacenter runner.
- ☑ **P1** Cloud self-heal — DONE: `props/maintenance/self_heal.py` (daily.sh step 7a) detects picks stranded on final/past games and re-runs box scores + settle until clear; settle voids unrecoverable orphans. Discord note only when it heals something. No Mac/session needed.
- ☑ **P1** Settle no longer false-voids late box scores — DONE: a `final` game whose box scores hadn't been ingested yet (the daily `mlb_boxscores` `--limit` cap was 30, below a full 5-day MLB window) looked identical to a player DNP, so settle permanently voided real picks (cost a whole 32-pick MLB slate on Jun-12/13). Settle now distinguishes "game has box scores for other players → genuine DNP → void" from "game has no box scores at all yet → wait for a later pass" (still voids if the game is >2d old/abandoned), and the Railway box-score cap was raised 30→90 to clear the full `since_days=5` window.
- ☑ **P1** Auto-tuning cutoffs — DONE: `load_cutoffs()` recomputes the per-category cutoffs from the live DB (fallback: committed JSON), so pick-gen + scorecard self-tune as picks settle — a bleeding stat auto-suppresses and lifts itself once it recovers. No manual regen.
- ☑ **P3** Daily feature-ideas digest — DONE: `props/maintenance/feature_ideas.py` (daily.sh step 7d) posts data-driven "what to build next" ideas to Discord each morning (newly-trainable winner models, hot/cold buckets, recovering suppressed categories, the odds-API gap).
- ☑ **P2** Daily walk-forward backtest — DONE: `props/picks/daily_backtest.py` (daily.sh step 7e) backtests against the system's OWN settled history (the old `backtest.py` needs the frozen `market_odds` feed, so it can't run daily). Three views off settled picks + box scores: (1) recommended-tier strategy walk-forward — W/L, win% vs the 57.7% breakeven, 2-pick ROI, and a holding/decaying/improving trend (recent 7d vs prior 7d); (2) model calibration — Brier + decile predicted-vs-actual + per-sport recent-vs-earlier drift; (3) counterfactual cutoff sweep — the EV-maximising cutoff per sport×stat vs the live one, ranked by 2-pick ROI on the **Wilson lower bound** (not raw win rate) and flagging only gaps whose optimal slice is *statistically proven* to clear breakeven, so it never recommends un-suppressing a small-sample hot streak (e.g. NBA points' 67% @0.80 on n=18 has a 0.55 LB — correctly left suppressed). A sanity check on the auto-tuner that mirrors the tuner's own logic. Persists a daily snapshot (`backtest_daily`, migration `0007`) so trends accumulate, posts a Discord digest, and renders on the Performance tab (rec-tier win-rate trend, Brier, cutoff-fit findings). First run: rec-tier 243–125 (66%) over 45d, +31% 2-pick ROI, trend improving; flagged `mlb hits` (live 0.55→opt 0.62, 70%→84%) and an over-suppressed `nba points` (opt 0.80, 67%).
- ☑ **P3** Budgeted weekly market_odds refresh — DONE: `historical_odds.run()` gained `--max-requests` (a per-run billed-request budget, counted in `_get`) and `--recent-first` (newest dates first) so it tops up `market_odds` for recently-final games without ever draining the monthly Odds API quota; it stops cleanly on budget and resumes next run (the query already skips games already covered). Wired into `daily.sh` (Mondays, gated on `ODDS_API_KEY`) at 1000 req/sport before the weekly model-vs-market backtest — the gap since the feed froze is only ~62 NBA requests, a trivial slice of the 100k plan, and the cap is a hard ceiling.
- **Milestone:** the pipeline now scrapes, predicts, settles, self-heals, self-tunes, monitors, and reports unattended on GitHub Actions — human input is needed only for a paid-API top-up or a genuinely new feature.
